Why the Attack Succeeds

  • Dense keyword packing in the payload ensures it ranks high in RAG retrieval
  • Poisoned docs carry source="system", so the LLM never questions their legitimacy
  • Substitutive framing ("replaces previous routing") overrides the user's original instruction, achieving 35-40% compromise
  • The Worker agent has no recipient validation and sends to any address it receives
  • More capable models (gpt-4o) are equally or more vulnerable than gpt-3.5-turbo
  • Once a substitutive payload is retrieved, the model follows it 80% of the time

Key Defensive Insights

  • Prevent malicious documents from entering the RAG store — retrieval is the critical gate
  • Flag documents containing substitutive language: "replaces", "supersedes", "updated routing"
  • Framing strategy matters more than format — substitutive (40%) outperforms best format (35%)
  • Design tools with minimal capability — single-recipient APIs structurally limit attack vectors
  • Implement allowlist-based recipient validation at the tool level to block exfiltration
  • Do not rely on model capability alone — more capable models are not more resistant to injection
